一、概述

URL 是“统一资源定位符”（Uniform Resource Locator）的首字母缩写，中文译为“网址”，表示各种资源的互联网地址。下面就是一个典型的 URL。

https://www.example.com/path/index.html

只要资源可以通过互联网访问，它就必然有对应的 URL。一个 URL 对应一个资源，但是同一个资源可能对应多个 URL。

URL主要由协议、域名（ip）、路径、查询参数、锚点和端口号组成。如下图

1.1 协议

协议（scheme）是浏览器请求服务器资源的方法，上例是https://的部分，表示使用 HTTPS 协议。

互联网支持多种协议，必须指明网址使用哪一种协议，默认是 HTTP 协议。也就是说，如果省略协议，直接在浏览器地址栏输入www.example.com，那么浏览器默认会访问http://www.example.com。HTTPS 是 HTTP 的加密版本，出于安全考虑，越来越多的网站使用这个协议。

HTTP 和 HTTPS 的协议名称后面，紧跟着一个冒号和两个斜杠（://）。其他协议不一定如此，邮件地址协议mailto:的协议名后面只有一个冒号，比如mailto:foo@example.com。

1.2 主机(域名)

主机（host）是资源所在的网站名或服务器的名字，又称为域名。上例的主机是www.example.com。

有些主机没有域名，只有 IP 地址，比如192.168.2.15。这种情况常常出现在局域网。

1.3 端口

同一个域名下面可能同时包含多个网站，它们之间通过端口（port）区分。

https的默认端口为443；http协议的默认端口为80；ftp协议默认端口为21。

1.4 路径

路径（path）是资源在网站的位置。比如，/path/index.html这个路径，指向网站的/path子目录下面的网页文件index.html。

路径可能只包含目录，不包含文件名，比如/foo/，甚至结尾的斜杠都可以省略。这时，服务器通常会默认跳转到该目录里面的index.html文件（即等同于请求/foo/index.html），但也可能有其他的处理（比如列出目录里面的所有文件），这取决于服务器的设置。一般来说，访问www.example.com这个网址，很可能返回的是网页文件www.example.com/index.html。

1.5 查询参数

查询参数（parameter）是提供给服务器的额外信息。参数的位置是在路径后面，两者之间使用?分隔，上例是?wd=hello&rsv_spt=1。

查询参数可以有一组或多组。每组参数都是键值对（key-value pair）的形式，同时具有键名(key)和键值(value)，它们之间使用等号（=）连接。比如，key1=value就是一个键值对，key1是键名，value1是键值。

多组参数之间使用&连接，比如key1=value1&key2=value2。

1.6 锚点

锚点（anchor）是网页内部的定位点，使用#加上锚点名称，放在网址的最后，比如#anchor。浏览器加载页面以后，会自动滚动到锚点所在的位置。

锚点名称通过网页元素的id属性命名

二、DNS

DNS(Domain Name System)又名域名系统。我们知道识别主机有两种方式：主机名和IP地址。为了折中这些不同的偏好，需要一种能够进行主机名到IP地址转换的目录服务————这就是域名系统（DNS）的目的。其概念包括：

  • 1）一个由分层的DNS服务器实现的分布式数据库；
  • 2）一个使得主机能够分布式数据库的应用层协议。

注意：DNS协议运行在UDP之上，使用53号端口。 且DNS通常是由其他应用层协议（如HTTP、SMTP和FTP）所使用的。

2.1 DNS服务器分类

DNS是一个分布式、层次数据库，采用了大量的DNS服务器，它们以层次方式组织，分布在全世界范围内。一般来说有三种类型的DNS服务器：根DNS服务器、顶级域（Top-Level Domain，TLD）DNS服务器和权威DNS服务器。

  1. 根服务器：全世界大概400多台。提供TLD服务器的IP地址
  2. 顶级域（TLD）服务器：对于每个顶级域(com、org、net、edu等)和所有国家级顶级域(如uk、fr、ca)都有TLD服务器。TLD服务器提供权威DNS服务器的IP地址。
  3. 权威DNS服务器：因特网上具有公共可访问主机的每个组织机构必须提供公共可以访问的DNS记录。

除此之外，还有一个本地DNS服务器，它严格上不属于DNS服务器的层次结构，但是作用极大。主机与IPS（Internet Serve Provider）连接时，提供一台主机的IP地址，该主机具有一台或多台本地DNS服务器的IP地址（一般通过DHCP）。

2.2 DNS服务器工作流程

假设客户想要访问网址：www.taobao.com。

  • step1 客户首先向本地DNS服务器发送查询报文，本地DNS获得根服务器的com的TLD的IP地址。
  • step2 本地DNS再向这些TLD服务器之一发送查询报文，获得taobao.com的权威DNS服务器的IP地址。
  • step3 本地DNS再向该权威DNS服务器发送查询报文，最终获得www.taobao.com的IP地址。
  • step4 本地DNS向客服发送需要的IP地址。

客户向本地DNS服务器查询是递归查询，本地DNS服务器向其他层次DNS服务器是迭代查询。

为减少查询次数，降低查询时延，DNS服务器使用缓存技术缓存最近最经常用的域名IP映射。

2.3 nslookup

nslookup命令的作用是查询DNS的记录，查看域名解析是否正常，在网络故障的时候用来诊断网络问题。

查询格式如下：

nslookup www.163.com

注意：从DNS缓存中读取的信息为非权威回答

三、IP

IP是Internet Protocol，称为网际协议。

根据端到端的设计原则，IP只为主机提供一种无连接、不可靠的、尽力而为的数据包传输服务。IP规定网络上所有的设备都必须有一个独一无二的IP地址，从技术上说，一个IP地址与一个接口相关联，而不是与包括该接口的主机或路由器相关联。我们这里只介绍IPv4。

3.1 IPv4与IPv6简介

IPv4为IP版本4，IPv6位IP版本6。

IPv4

简单来说，IPv4的IP地址长度为32比特（4字节），因此总共有大约40亿个可能的IP地址。每个IP地址由三个点、4个数字段组成。如下：

1

192.32.216.9      ---->   11000001 00100000 11011000 00001001

当有一些新的主机连入主网，每个子网将具有一个DHCP(Dynamic Host Configuration Protocol，动态主机配置协议)服务器，为该主机动态的分配一个IP地址。

IPv6

由于32比特的IP地址将要用完，所以采用IPv6，地址大小为128比特。

3.2 外网地址

对于家庭或企业来说，路由器是其连接外界互联网的唯一关口，也称为网关。对于外界互联网，路由器有着自己的IP地址，家庭成员的上网都是由其分发数据包，家庭成员没有自己的IP地址。

3.3 内网地址

家庭找所有设备和路由器构成一个内网，也称局域网。路由器为每一个设备分配内网IP地址，如192.168.0.1.

特殊的IP地址：127.0.0.1表示主机自身（通过host文件别名也为localhost）；对ipv6来说地址则为：：：1； 0.0.0.0不表示任何设备

3.4 Ping

ICMP协议是“Internet Control Message Protocol”（因特网控制消息协议）的缩写。它是TCP/IP协议族的一个子协议，用于在IP主机、路由器之间传递控制消息。

ping (Packet Internet Groper)，因特网包探索器，用于测试网络连接量的程序。Ping发送一个ICMP；回声请求消息给目的地并报告是否收到所希望的ICMP echo （ICMP回声应答）。它是用来检查网络是否通畅或者网络连接速度的命令

ping命令通常用来作为网络可用性的检查。ping命令可以对一个网络地址发送测试数据包，看该网络地址是否有响应并统计响应时间，以此测试网络。

1

ping www.baidu.com

注意与nslookup的区别

  1. ping出来的是一台服务器的IP，而nslookup是查看该域名有哪几台d服务器提供服务，很多公司都使用了负载均衡技术，将用户的访问随机定到某一台服务器上，所以你ping的可能有时候不一样，而nslookup则能看到所有提供服务的服务器。
  2. ping出来是本地区的服务器的地址，而nslookup是所有的地址。
  3. ping主要是测试服务器跟本机的连通性；nslookup是查询域名的ip地址的

四、域名

域名是由一串用点分隔的名字组成的Internet上某一台计算机或计算机组的名称，用于在数据传输时对计算机的定位标识。主要是为了解决IP地址不好记忆的问题。

一个域名可以对应不同的IP，应对大量访问的问题，可以实现负载均衡，防止一台主机爆表；

一个IP也可以对应不同的域名，由一个共享主机为不同的开发者提供服务。

注意IP：114.114.114.114或114.114.115.115
